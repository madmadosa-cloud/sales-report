매출 분석 프로그램

ERP 판매현황 CSV에서 거래처 분류집계 · 품목 분류집계 · 직납 품목 집계, 그리고 거래처별 구매현황 지역분류를 자동 생성합니다

데이터 파일 불러오기

이카운트 등 ERP에서 내려받은 판매현황 · 구매현황 CSV를 불러온 뒤 [분석 실행]을 누르세요. 여러 달이 한 파일에 있어도 월별로 나눠 분석·저장합니다. 분석한 CSV·월별 데이터는 프로그램 폴더(저장데이터)에 자동 보관되어 다시 켜도 유지됩니다.

선택된 파일 없음
선택된 파일 없음
직납 기준 = (1) 품목코드에 '직납'이 포함되고 (2) 품목명에 아래 [직납 인식 브랜드] 중 하나가 포함된 경우
구매현황 지역분류 = 거래처코드가 z4~z6 이면 '경상남도외 생산시설', 그 외(a~z3)는 '경상남도 생산시설'
판매 CSV는 품목코드·거래처코드·수량·합계가 한 행에 있는 품목별·거래처별 형식입니다. 여러 달은 일별(또는 일자) 열로 월별 자동 분리됩니다.

저장된 월별 데이터

[분석 실행] 시 월별로 자동 저장됩니다. 연도·월을 선택해 그 달 결과를 보거나, [1년 한눈에 보기]로 해당 연도 1~12월을 한 표에서 비교할 수 있습니다.

아직 저장된 월이 없습니다. CSV를 불러온 뒤 [분석 실행]을 누르세요.

엑셀 다운로드 — 화면에 보이는 결과, 또는 저장된 월별 데이터를 종류별 1시트(1월·2월… 열)로 받을 수 있습니다. 저장 후 다운로드 폴더가 자동으로 열립니다.

분류코드 관리

품목·거래처 분류코드를 직접 입력하거나 CSV로 일괄 등록합니다. 변경 내용은 이 PC에 저장되어 다음 실행 시에도 유지됩니다.

품목분류코드

코드는 영문 소문자 1글자(a~z). 보고서 품목 행 순서는 코드 순서를 따릅니다.

코드 분류명 사용 데이터 건수 관리

CSV 일괄 업로드

품목및거래처분류코드.csv 형식 — 품목코드 구간만 사용됩니다.

거래처분류코드

코드는 영문 소문자 1글자(a~z). 보고서 거래처 구간 순서는 코드 순서를 따릅니다.

코드 분류명 사용 데이터 건수 관리

CSV 일괄 업로드

품목및거래처분류코드.csv 형식 — 거래처코드 구간만 사용됩니다.

직납 인식 브랜드

품목명에 아래 브랜드 중 하나가 들어있고 품목코드에 '직납'이 있으면 직납으로 집계됩니다. 브랜드를 추가/삭제하면 이 브라우저에 저장되어 다음부터 자동으로 적용됩니다.

기간 비교 분석 보고서

저장된 월별 데이터를 묶어 비교 보고서를 만듭니다. 기준·비교 기간 사이의 모든 연도·월도 함께 포함합니다. 예) 기준 2026년 1~5월 vs 비교 2023년 1~5월 → 2024·2025년 데이터 포함

오픈소스·라이선스 고지

앱 버전 1.0.0

본 프로그램은 오픈소스 라이브러리를 사용합니다. 법적 효력·최신 상태는 각 프로젝트의 공식 라이선스 전문을 우선합니다. 배포 폴더의 오픈소스_라이선스_고지.txt 와 동일 내용입니다.

1. 응용 프로그램 소스 코드

매출분석.html, app_launcher.py 등 본 프로그램의 응용 소스 코드에 대한 권리는 개발·보유 주체에 귀속됩니다. 무단 복제·배포를 금합니다.

2. 화면(프론트엔드) JavaScript

분석 화면·차트(SVG)·표 렌더링은 외부 CDN·npm 라이브러리 없이 순수 JavaScript로 작성되었습니다. Chart.js, jQuery, React 등은 사용하지 않습니다.

4. 운영 체제·런타임

  • Microsoft Edge WebView2 — Windows에서 화면 표시 (OS 기본 포함 또는 별도 런타임)
  • Microsoft Windows — 운영 체제

5. 개발 PC 전용 (배포 PC 불필요)

앱빌드.bat 실행 시 PyInstaller·pip를 사용합니다. 다른 PC에는 Python 설치가 필요 없습니다.
